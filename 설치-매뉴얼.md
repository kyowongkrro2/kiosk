# 학습개발2팀 키오스크 — 설치 매뉴얼

이 문서는 **처음 설치하는 사람(새 관리자)** 을 위한 설명서입니다.
파일 2개(index.html, Code.gs)만 있으면, 본인의 구글·깃허브 계정으로 약 30분 만에
팀 커피 주문 키오스크를 구축할 수 있습니다. 전임자에게 계정이나 주소를 받을 필요가 없습니다.

## 준비물

- **index.html, Code.gs** 파일 2개 (전임자에게 받거나 기존 저장소에서 다운로드)
- 본인 **구글 계정** (데이터 저장용)
- 본인 **깃허브 계정** (화면 공개용 — 없으면 github.com에서 무료 가입. 아이디가 팀 주소의 일부가 되니 신중히)

## 완성 후 구조

| 조각 | 역할 | 위치 |
|---|---|---|
| 깃허브 저장소 | 화면(index.html) 보관 + 팀 공개 | 내 깃허브 |
| 앱스 스크립트 | 데이터 저장/불러오기 서버 | 내 구글 계정 |
| 스프레드시트 | 데이터 저장소 (자동 생성됨) | 내 구글 드라이브 |

# 1단계. 데이터 서버 만들기 (구글)

1. 구글에 로그인한 상태로 주소창에 `script.new` 입력
2. 왼쪽 위 "제목 없는 프로젝트" → 이름을 "팀 키오스크"로 변경
3. Code.gs 기본 내용을 전부 지우고, 받은 **Code.gs 파일 내용을 통째로 붙여넣기** → 저장(Ctrl+S)
4. 오른쪽 위 **배포 → 새 배포**
5. 톱니바퀴 → **웹 앱** 선택 ⚠️ ("라이브러리" 아님 — 가장 흔한 실수)
6. 실행: **나** / 액세스: **모든 사용자** ⚠️ (다르게 하면 저장이 안 됩니다)
7. **배포** → 권한 승인. "Google hasn't verified this app" 경고는 정상입니다:
   **Advanced → Go to (프로젝트명) → Continue(허용)**
8. 발급된 **웹 앱 URL** (`https://script.google.com/macros/s/…/exec`) 복사해 두기

확인: 새 탭에서 그 URL을 열면 "API 서버 동작 중입니다."가 보여야 합니다.

# 2단계. 화면 파일 설정 (index.html)

받은 index.html을 메모장(또는 아무 편집기)으로 열어 **두 군데**만 수정:

1. `API_URL` 검색 → 따옴표 안을 1단계에서 복사한 웹 앱 URL로 교체
   ```
   const API_URL = "https://script.google.com/macros/s/…/exec";
   ```
2. `ADMIN_PIN` 검색 → 관리자 비밀번호를 원하는 번호로 설정
   ```
   const ADMIN_PIN = "0000";
   ```

저장. 파일명은 반드시 `index.html` 그대로.

# 3단계. 팀에 공개하기 (깃허브)

1. 깃허브 로그인 → 오른쪽 위 **＋ → New repository**
2. 이름 입력(팀 주소 뒷부분이 됨, 예: kiosk) → **Public** 선택 → Create repository
3. **uploading an existing file** 클릭 → 수정한 **index.html**과 **Code.gs**(백업용), 이 매뉴얼을 끌어다 놓기 → **Commit changes**
4. 저장소 **Settings → Pages** → Source: Deploy from a branch / Branch: **main**, **/(root)** → Save
5. 1~2분 뒤 페이지 새로고침 → "Your site is live at …"에 표시된 주소가 **팀 공유 주소**입니다
   (예: `https://내아이디.github.io/kiosk/`)

# 4단계. 검증 체크리스트

- [ ] 팀 주소 접속 → 키오스크 화면이 뜬다 (흰 화면이면 Ctrl+Shift+R 강력 새로고침)
- [ ] 테스트 주문 1건 입력 → 오른쪽 위 **"저장됨"** 표시
- [ ] 내 구글 드라이브에 "학습개발2팀 키오스크 데이터" 시트가 자동 생성되고 주문내역 탭에 기록됨
- [ ] 카페 관리 탭 🔒 → 내가 정한 비밀번호로 열림
- [ ] 테스트 주문은 집계 탭에서 관리자 모드(이름 옆 ×)로 삭제

전부 통과하면 설치 완료. 팀 채팅방에 주소를 공유하세요.

# 5단계. 초기 설정 (앱 안에서)

첫 설치 시 기본 팀원 명단·카페 3~4곳·예시 주문이 들어 있습니다. 앱에서 정리하세요:

- **팀원 관리** 🔒: 명단·생일 수정, 퇴사자 삭제, 휴직 처리
- **카페 관리** 🔒: 이번 달 카페 지정, 안 가는 카페 삭제, 메뉴·가격 수정
- **집계**: 필요 없는 예시 주문은 × 로 삭제

# 운영 안내

### 일상 관리 (코드 수정 불필요)
매달 하는 일은 앱 안에서 끝납니다: 카페 지정 → 팀원들이 주문 → 집계 탭에서 "주문 내용 복사" → 카페에 전달.
한 달에 여러 번이면 카페 관리에서 차수 추가. 집계는 구글 시트(대시보드/주문내역 탭)에도 자동 정리됩니다.
⚠️ 시트의 `_data` 탭은 절대 직접 수정 금지. 다른 탭도 보기 전용(고쳐도 덮어써짐).

### 기능 수정 — 클로드(claude.ai) 활용
이 앱은 클로드로 만들어졌고, 수정도 같은 방법으로 합니다:
1. 저장소에서 index.html 다운로드 → claude.ai 새 대화에 업로드
2. 요청 예: "우리 팀 생일 커피 주문 키오스크야(리액트, 깃허브 페이지, 구글 앱스 스크립트 저장). ○○ 수정해줘."
   새 카페 추가는 메뉴판 사진을 함께 첨부.
3. 받은 파일을 저장소에 덮어쓰기 업로드 → 1~2분 뒤 반영

### 서버(Code.gs) 수정 철칙
수정 후 반드시: 배포 → **배포 관리** → 연필 → 버전 **"새 버전"** 선택 → 배포
⚠️ "새 버전"을 안 고르면 아무리 고쳐도 반영 안 됨 / ⚠️ "새 배포"를 누르면 URL이 바뀌어 앱이 끊김

### 문제 해결
| 증상 | 해결 |
|---|---|
| 화면이 흰색 | Ctrl+Shift+R → 안 되면 시크릿 창 → 그래도면 F12 콘솔의 빨간 오류 확인 |
| "저장 실패" | 웹앱 URL 뒤에 `?action=load` 붙여 접속해 진단: 로그인 화면이 뜨면 → 배포 액세스를 "모든 사용자"로 / 오류 문구가 뜨면 → 새 버전 재배포 |
| 수정했는데 반영 안 됨 | 화면은 깃허브 재업로드, 서버는 새 버전 재배포 |
| 시트 정리 탭이 안 바뀜 | 앱스 스크립트에서 함수 `수동갱신` 선택 → 실행 |

### 다음 인수인계
이 방식 그대로 반복하면 됩니다: 후임자에게 **index.html, Code.gs, 이 매뉴얼**을 넘기면,
후임자가 본인 계정으로 새로 설치합니다. 팀 주소가 바뀌므로 설치 후 팀에 새 주소를 공지하고,
전임자는 본인의 옛 저장소와 앱스 스크립트를 삭제하면 됩니다.
